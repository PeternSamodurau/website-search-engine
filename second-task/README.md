### Приложение «Учёт студентов» 

#### Этап 1: Настройка проекта и создание основной сущности
Подготовка основы проекта и определение модели данных студента.

*   **`build.gradle`** Добавление зависимостей:
    *   `spring-boot-starter`
    *   `spring-boot-starter-shell` (версии 3.2.5)
    *   `lombok`
*   **`Student.java`**: Создание класса-сущности `Student` с полями `id` (UUID), `firstName` (String), `lastName` (String), `age` (int). 
        Использование аннотаций Lombok (`@Data`, `@AllArgsConstructor`) для автоматической генерации `getters`, `setters`, `toString` и конструкторов.

#### Этап 2: Создание слоя хранения данных (Repository)
Создание компонента для хранения студентов в памяти (в коллекции `Map`, как рекомендовано).

*   **`StudentRepository.java`**: Создание интерфейса, описывающего операции с хранилищем: `findAll()`, `save()`, `deleteById()`, `deleteAll()`.
*   **`InMemoryStudentRepository.java`**: Создание реализации интерфейса. Использование `java.util.Map<Long, Student>` для хранения данных.

#### Этап 3: Создание слоя бизнес-логики (Service)
Создание центрального сервиса для управления логикой и публикации событий.

*   **`StudentService.java`**: Создание интерфейса для сервиса: `getAllStudents()`, `addStudent()`, `deleteStudent()`, `clearStudents()`.
*   **`StudentServiceImpl.java`**: Создание реализации сервиса. Использование `StudentRepository` для доступа к данным и `ApplicationEventPublisher` для публикации событий.

#### Этап 4: Реализация системы событий
Реализация механизма для реакции на создание и удаление студентов через `EventListener`.

*   **`StudentCreatedEvent.java`**: Создание класса-события, хранящего информацию о созданном студенте.
*   **`StudentDeletedEvent.java`**: Создание класса-события, хранящего `id` удалённого студента.
*   **`StudentEventListener.java`**: Создание компонента-слушателя с двумя методами, помеченными `@EventListener`:
    *   Один метод для обработки `StudentCreatedEvent` (выводит в консоль информацию о созданном студенте).
    *   Второй метод для обработки `StudentDeletedEvent` (выводит в консоль `id` удалённого студента).

#### Этап 5: Создание пользовательского интерфейса (Spring Shell)
Реализация возможности управлять приложением через консольные команды.

*   **`StudentCommands.java`**: Создание `@ShellComponent` с методами, помеченными `@ShellMethod`:
    *   `list`: для вывода всех студентов (команда просмотра).
    *   `add`: для добавления нового студента (команда сохранения). Принимает `firstName`, `lastName`, `age`.
    *   `delete`: для удаления студента по `id` (команда удаления).
    *   `clear`: для полной очистки списка студентов (команда очистки).

#### Этап 6: Настройка начальной загрузки данных
Добавление отключаемой функции создания студентов при старте приложения.

*   **`application.properties`**: Добавление свойства `app.students.initialize=true` для управления функцией.
*   **`StudentInitializer.java`**: Создание компонента-слушателя, который реагирует на событие `ApplicationReadyEvent`. Компонент будет помечен аннотацией `@ConditionalOnProperty(name = "app.students.initialize", havingValue = "true")`, чтобы он активировался только при наличии и значении `true` указанного свойства. При срабатывании он добавит нескольких студентов для демонстрации.

#### Этап 7: Контейнеризация и документирование
Упаковка приложения в Docker-контейнер и написание инструкции по его использованию.

*   **`Dockerfile`**: Создание файла с инструкциями для сборки Docker-образа. Будет использована многоэтапная сборка для оптимизации размера и безопасности образа.
*   **`README.md`**: Написание документации с описанием:
    *   Процесса запуска (локально и через Docker).
    *   Настроек по умолчанию и способов их переопределения (например, через переменные окружения при запуске контейнера).
    *   Списка доступных команд и их использования.
