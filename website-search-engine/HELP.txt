1.SiteInitializationService.java (папка init):
•Проблема: Сервис при запуске приложения безусловно устанавливает всем сайтам статус INDEXING, даже если реальный процесс индексации не запускается.•Нарушение ТЗ: Это вводит в заблуждение и противоречит семантике статусов, описанной в ТЗ, где INDEXING должен отражать активный процесс обхода сайта.2.IndexRepository.java (папка repository):•Проблема: Отсутствует эффективный метод для удаления всех записей из поискового индекса (index table), связанных с одной конкретной страницей (например, deleteAllByPage(Page page)).•Нарушение ТЗ: Это заставляет другие сервисы (LemmaServiceImpl) использовать менее производительный обходной путь (получение всех записей и удаление их по одной или списком) для реализации требования по переиндексации отдельной страницы.3.SearchServiceImpl.java (папка services):•Проблема: Метод generateSnippet при создании фрагмента текста для результатов поиска может начать этот фрагмент с середины слова.•Нарушение ТЗ: Это ухудшает читаемость сниппета и не полностью соответствует требованию о предоставлении качественного, понятного для пользователя результата поиска.4.SiteCrawler.java (папка services):•Проблема (Критическая): Полностью отсутствует какая-либо логика для загрузки, парсинга или учета правил из файла robots.txt.•Нарушение ТЗ: Это является прямым и грубым нарушением явного требования ТЗ: "При обходе страниц необходимо соблюдать правила, описанные в файле robots.txt".5.StatisticsServiceImpl.java (папка services):•Проблема: Логика сбора статистики некорректна. Общая статистика (total) подсчитывается отдельно от детальной (detailed), что может привести к расхождению данных (сумма по detailed не будет равна total).•Нарушение ТЗ: ТЗ неявно предполагает консистентность и точность предоставляемых данных, а текущая реализация может вводить пользователя в заблуждение.
